A Common Lisp dialect
that fits in your head.

ren is five reader-macros, thirty lines of code, and one rule: the subject is implicit. 

;; methods read like prose — `i` is self, `$x` is its slot
(defmethod add ((i num) v)
  (incf $n)
  (let ((d (- v $mu)))
    (incf $mu (/ d $n))
    (incf $m2 (* d (- v $mu)))))

;; chain through structures, look up config, bind `it`
(when (< (? data cols n) !budget)
  (aif (next-row pool)
       (label it)))

Why ren?

Tiny

The whole language is one file: a reader-macro definer plus four reader macros. Drop it in, (load "lib"), and you're done.

Anaphoric

In a method, i is always self. In an aif, it is always the test. No more boilerplate let-bindings.

Composable

$, !, and ? read as Lisp forms. They setf, they incf, they nest — they're invisible until you look.

Try it

Two files. No build system, no package manager, no asdf:load-system. If you have CLISP or SBCL, you have ren.

$ git clone https://github.com/timm/lisp
$ cd lisp
$ sbcl --script fri2.lisp --stats
$ clisp fri2.lisp --tree

The five macros

!key

Config lookup.

Reads from *the*, an alist of (name value flag doc) tuples. The flag and doc serve double duty as the command-line spec.

!budget   ;; ≡   (second (assoc 'budget *the*))

$field

Self slot access.

Inside a method whose first parameter is named i, $field expands to (slot-value i 'field). Works with setf, incf, and friends.

(defmethod mid ((i num)) $mu)

(defmethod add ((i sym) v &optional (w 1))
  (unless (eql v '?)
    (incf $n w)))

(? x a b c)

Nested slot access.

Chains slot-value calls. setf-able. Replaces the noisy (slot-value (slot-value ...)) tower.

(? data cols y)
;; ≡
(slot-value (slot-value data 'cols) 'y)

(aif test then else)

Anaphoric if.

Binds the test result to it in both branches. Borrowed from Paul Graham's On Lisp.

(aif (find flag opts :key #'third :test #'equalp)
     (setf (second it) (thing arg)))

(defread c (s) body)

Reader-macro definer.

Installs a one-character reader macro. The other four macros are all built with this. Six lines of code, and the language extends itself.

(defread $(s)
  `(slot-value i ',(read s t nil t)))

(defread !(s)
  `(second (assoc ',(read s t nil t) *the*)))

Cookbook

Five recipes lifted straight from fri2.lisp — a 450-line active-learning system written in ren.

1. Polymorphic stats with one symbol per slot

The same mid generic returns the mode of a symbolic column, the mean of a numeric one, or a row of per-column mids for a whole dataset. Each method reads only the slots it cares about — via $.

(defmethod mid ((i sym))
  (labels ((most (a b) (if (> (cdr a) (cdr b)) a b)))
    (car (reduce #'most $has))))

(defmethod mid ((i num)) $mu)

(defmethod mid ((i data))
  (or $mid
      (setf $mid (mapcar #'mid (? i cols all)))))

No with-slots, no (slot-value self ...) tower, no accessor names to invent.

2. A constructor that inspects itself

Bind i in &aux, and the body of the constructor can use $ just like a method. Here, a numeric column reads its own header text to decide whether the goal is to minimize or maximize.

(defun make-num (&rest args
                 &aux (i (apply #'%make-num args)))
  (setf $goal (if (eq #\- (ch $txt -1)) 0 1))
  i)

3. Command-line flags meet config in one line

Each entry in *the* carries its flag and doc right next to its value. The CLI walks argv, dispatches --cmds to eg- functions, and uses aif to update any matching -flag.

(defparameter *the*
  '((budget 50  "-b" "max labels")
    (few    128 "-u" "max unlabelled pool")))

(defun cli (lsts)
  (loop for (flag arg) on (args) by #'cddr do
    (unless (run flag (thing arg))
      (aif (find flag lsts :key #'third :test #'equalp)
           (setf (second it) (thing arg))))))

The it in (second it) is the matched *the* entry — bound only if find succeeded.

4. A hot loop that reads structures and config

The active-learning inner loop walks a pool, stops when the label budget is hit, and asks each candidate whether it sits closer to best than rest. ? chains the slot lookup, ! reads the budget — both compile to ordinary Lisp.

(loop for r in (subseq pool 0
                       (min !few (length pool)))
      while (< (? ys n) !budget) do
      (when (closer? lab best rest r)
        (add ys (disty lab r))
        (add lab r) (add best r)
        (rebalance best rest lab)))

5. Implicit i outside of methods

The $ reader macro just expands to (slot-value i ...) — it doesn't care whether i came from a method dispatch or a lambda parameter. disty exploits this to compute Minkowski distance over goal columns:

(defun disty (data row &optional (cols (? data cols y)))
  (dist cols
    (lambda (i)
      (abs (- (norm i (elt row $at)) $goal)))))

Inside the lambda, i is each column in turn — and $at, $goal work without ceremony.

A complete example

Here is a streaming numeric summary — mean and standard deviation in constant memory, using Welford's method — written in plain Common Lisp on the left, and in ren on the right.

Plain Common Lisp

(defmethod add ((self num) v)
  (incf (slot-value self 'n))
  (let ((d (- v (slot-value self 'mu))))
    (incf (slot-value self 'mu)
          (/ d (slot-value self 'n)))
    (incf (slot-value self 'm2)
          (* d (- v (slot-value self 'mu))))))

ren

(defmethod add ((i num) v)
  (incf $n)
  (let ((d (- v $mu)))
    (incf $mu (/ d $n))
    (incf $m2 (* d (- v $mu)))))

Same code, half the noise, no new concepts. ren doesn't hide Lisp — it gets out of the way.
