A Tour of fri2.lisp — the first three steps

From one lonely atom, to a pair of rows, to two distributions facing off. Same arc as the Python tour; this time in Common Lisp; smallest code that earns the words “that’s odd” and proposes a delta.

A value sits alone. It does not know if it is large or small. We give it a column — some friends to compare with — and now it can be surprised. We give the column a partner — another column, in another row — and now two rows can disagree. We give the rows siblings — two heaps of them — and now two distributions can argue about where the boundary should sit. Three steps. Three deltas.
Where we end up 
(defparameter *the* '((p 2 "-p" "minkowski") ...))
(defstruct (num (:constructor %make-num))

A num remembers a column in constant space. A data is a list of rows plus one num per column. Distance, surprise, splits — all of them lean on this one struct.

1. A lonely atom

Type something. Anything.

* 42
42
* '?
?
* (list 42)
(42)

The REPL evaluates anything. Atom, symbol, list. Not just function calls.

What can we say about 42 alone?

Nothing. Is 42 big? Small? Typical? The atom is a dot without an axis. Surprise has no meaning until there are companions to be surprised against.

The smallest companion: a Num

To call a value odd we need two scalars: a centre µ and a width σ. Welford’s 1962 trick stores both in constant space, updated one sample at a time, no list of past values kept.

  μₙ  = μₙ₋₁ + (v − μₙ₋₁) / n
  M₂ₙ = M₂ₙ₋₁ + (v − μₙ₋₁) · (v − μₙ)
  σ   = √( M₂ / (n − 1) )

The matching Lisp lives in fri2.lisp at the (add ((i num) v &optional (w 1))) method:

(defmethod add ((i num) v &optional (w 1))
  (unless (eq v '?)
    (cond ((and (minusp w) (<= $n 2))
           (setf $n 0 $mu 0 $m2 0))
          (t (incf $n w)
             (let ((delta (- v $mu)))
               (incf $mu (float (/ (* w delta) $n)))
               (incf $m2 (float (* w delta (- v $mu))))))))
  v)
Note. $mu updates before $m2 reads it. Welford’s identity uses the new mean for the m2 term. Not a bug.

“That’s odd” — the smallest detector

With µ and σ in hand, the smallest code that earns the words “that’s odd” is a z-score:

  z = (v − μ) / σ

fri2 wraps this in norm, which clamps to ±3 (covering 99.7% of a Gaussian) and squashes through a sigmoid so the result reads as “fraction of column at or below v”:

(defmethod norm ((i num) v)
  (if (eq v '?) v
      (let ((z (/ (- v $mu) (+ (spread i) 1e-32))))
        (/ 1 (+ 1 (exp (* -1.7 (max -3 (min 3 z)))))))))

Try it on the Num we just built (µ = 30, σ ≈ 15.8):

* (norm n 30)        ; the mean
0.5
* (norm n 80)        ; way above
0.99
* (norm n -10)       ; way below
0.005
* (norm n 35)        ; mildly above
0.58

30 sits exactly at the median (0.5). 80 lands at the 99th percentile. −10 at the first. 35 is unsurprising — barely above middle. The detector says “that’s odd” whenever norm drifts toward 0 or 1.

The delta — what to do about it

Detection is not enough. The story needs a delta: a rule for what to do when something is odd. For one column the answer is the Welford increment itself.

  Δμ = w · (v − μ) / n
  &DeltaM;₂ = w · (v − μ_old) · (v − μ_new)

Two adds. The summary nudges toward the new value at rate 1/n. A surprising point pulls µ harder than a typical one, because (v − μ) is bigger. The column learns from the surprise. And, symmetrically, you can unlearn: pass w = −1 to add, or call (sub n v):

* (sub n 50) (sub n 40)
* (list $n $mu $m2 (spread n))
(3 20.0 200.0 10.0)

Right back where we were after step 3. This reversibility is what makes chapter 3’s comparison cheap.

Edge case. (add n '?) returns ? and touches no slot. Missing values propagate, never aggregate. Every later piece — norm, dist, tree — respects this rail.

2. Two rows

One value found its bearings inside one column. A row is many values across many columns. Once we have two rows, we can ask: are they near or far? — and if far, which columns drove it?

The smallest companion: a row

A row is just a list. The schema lives in the header.

* (defparameter rs (read-csv "auto93.csv"))
* (first rs)
("Clndrs" "Volume" "HpX" "Model" "origin" "Lbs-" "Acc+" "Mpg+")
* (second rs)
(8 304 193 70 1 4732 18.5 10)
* (car (last rs))
(4 86 65 80 3 2110 17.9 50)

Heavy V8 at row 1. Light four-cylinder at the end. A glance says these two rows are far apart. Now the smallest code to prove it.

distx — one number from many columns

Each x-column produces a per-cell gap. Roll up by Minkowski.

(defun dist (lst f &aux (d 0))
  (dolist (v lst (expt (/ d (length lst)) (/ 1 !p)))
    (incf d (expt (funcall f v) !p))))

(defmethod distx ((i data) r1 r2)
  (dist (? i cols x)
        (lambda (i)
          (let ((a (elt r1 $at)) (b (elt r2 $at)))
            (if (and (eq a '?) (eq b '?)) 1 (distx i a b))))))

(defmethod distx ((i sym) a b) (if (equal a b) 0 1))

(defmethod distx ((i num) a b
                          &aux (a (norm i a)) (b (norm i b)))
  (cond ((eq a '?) (abs (- b (if (> b 0.5) 0 1))))
        ((eq b '?) (abs (- a (if (> a 0.5) 0 1))))
        (t         (abs (- a b)))))

Three rules. Sym: same or different (0 or 1). Num: normalize, then absolute difference. Missing: push the unknown to the extreme furthest from the known — pessimistic, so the dimension still costs something.

* (defparameter d (make-data rs))
* (distx d (second rs) (second rs))           ; self
0.0
* (distx d (second rs) (third rs))            ; close twin
0.040
* (distx d (second rs) (car (last rs)))       ; opposite pole
0.821

“That’s odd” for two rows

0.0 is identity. 0.04 is unsurprising. 0.82 is the alarm bell. Just like the z-score, distx turns a vector difference into one scalar that we can compare to a threshold.

The delta — which column drove it

The roll-up hides the cause. Unfold one step and the per-column gaps come back as a list:

* (loop for c in (? d cols x)
        for a = (elt (second rs) (? c at))
        for b = (elt (car (last rs)) (? c at))
        collect (list (? c txt) a b (distx c a b)))
(("Clndrs" 8 4   1.0)
 ("Volume" 304 86 0.78)
 ("Model"  70 80  0.71)
 ("origin" 1 3    1.0))

Now the surprise is actionable. Two columns are maxed out: Clndrs and origin — categorical, no overlap. Volume and Model contribute another 0.7 each. If we were nudging row 2 toward row 1 to make it heavier, the deltas tell us which features to move first: cylinders, then origin, then volume.

Note. The distance does not use the y-columns. Lbs-, Acc+, and Mpg+ are withheld. We are measuring rows by their cheap features, so we can later predict their expensive labels.

3. Two distributions

One row could surprise inside a column. Two rows could disagree across columns. Now: two heaps of rows. Two data objects sitting side-by-side, each with its own per-column µ and σ. The question is no longer “is this point odd?” but “are these two populations the same?” — and if not, where is the boundary?

The smallest companion: split a Data in two

Sort by goal-distance. Top half is “best”, bottom half is “rest”.

* (defparameter ranked
    (sort (copy-list (? d rows)) #'< :key (lambda (r) (disty d r))))
* (defparameter half (floor (length ranked) 2))
* (defparameter best (make-data (cons (? d cols names)
                                       (subseq ranked 0 half))))
* (defparameter rest (make-data (cons (? d cols names)
                                       (subseq ranked half))))
* (length (? best rows)) (length (? rest rows))
199
199

Two distributions, materialized. Each data carries its own column summaries, ready to talk about themselves.

“That’s odd” for two distributions

The minimum code to know the heaps differ on a column is to ask each Num for its µ and compare:

* (loop for cb in (? best cols x) for cr in (? rest cols x)
        collect (list (? cb txt) (mid cb) (mid cr)))
(("Clndrs" 4.21  6.55)
 ("Volume" 122.3 234.8)
 ("Model"  77.1  74.5)
 ("origin" 2.10  1.40))

Best cars (low disty — light, efficient) average 4.2 cylinders. Rest average 6.5. Volume: 122 vs 235. The two populations live in different neighborhoods of feature space — that is the surprise.

The delta — one cut, the smallest

Knowing the heaps differ is not enough. We want a rule: one column, one threshold, that splits a fresh row into “probably best” or “probably rest”. Score every candidate cut by its weighted spread:

  score(cut) = n_left  · σ_left
             + n_right · σ_right

Lower score = cleaner split. fri2 picks the median of each numeric column as its candidate cut (regularization — more cuts overfit at small N) and scans every column:

(defun tree-split (d c cut rs)
  (let (l r (ln (make-num)) (rn (make-num)))
    (dolist (row rs)
      (if (go-left? c (elt row (? c at)) cut)
          (progn (push row l) (add ln (disty d row)))
          (progn (push row r) (add rn (disty d row)))))
    (list (+ (* (? ln n) (spread ln)) (* (? rn n) (spread rn)))
          c cut l r)))

Run it across every x-column on the labelled rows; pick the lowest-scoring cut.

* (defparameter splits (tree-splits d (? d rows) 3))
* (defparameter best-split (extremum-by splits #'first #'<))
* (list (? (second best-split) txt) (third best-split))
("Volume" 151)

One column, one number. Volume ≤ 151 is the smallest rule that separates “best” from “rest” with the lowest combined spread on the y-distance. That is the delta: do not just say the two distributions differ — say where the boundary lives, in a way a human can read aloud.

Note. The score is on disty, not on the column itself. We split a feature column, but we judge the split by how cleanly it separates goal outcomes. Cheap inputs, expensive labels — the same trade as chapter 12 of the Python tour.

From one cut to a tree

A single rule is the seed. Recurse on each side and you get the tree at tree-grow in fri2.lisp. Each leaf is a sentence. Each sentence is testable. The lonely atom of chapter 1, having found its column, its row, and its sibling distribution, has now produced a rule explainable in English.

Where we arrived

  atom    →  μ,σ       →  norm    →  “is v odd?”       →  Welford Δ
  row     →  per-col gap →  distx   →  “are r1,r2 odd?” →  per-col deltas
  heap    →  per-data Num→  spread  →  “heaps differ?”   →  one cut

Three iterations of the same shape: summarize, detect, propose a delta. Welford gives the column a memory. distx gives a pair of rows a verdict. The split-score gives two heaps a boundary. Each layer adds one structural fact — a column, a row, a heap — and gets one new noun in return: surprise, direction, rule.

The lonely atom now knows where it stands — and, given two heaps of its kin, can write down the line that tells them apart.
