# fri2: Data-Lite Active Learning

> One file. Reads a CSV, summarises columns in constant
> space, computes a goal distance, spends a tiny label
> budget, grows a small decision tree from those labels.
> Audience: newly arrived US-university grad students with
> basic programming. Target runtime: SBCL.

## Atoms

A column is symbolic or numeric. Two structs, paired and
parallel.

### sym

`sym` counts how often each value appears. `has` is an
a-list of `(value . count)` pairs. `at` is the column
index in a row; `txt` is the header string.

```lisp
(defstruct sym
  (at 0) (txt " ") (n 0) has)
```

### num

`num` keeps two scalars instead of every value: running
mean `mu` and second-moment accumulator `m2`. Welford
1962 derives `sd` on demand from `m2` and `n`. The
`heaven` slot records goal direction: 1 means "bigger is
better", 0 means "smaller is better".

```lisp
(defstruct (num (:constructor %make-num))
  (at 0) (txt " ") (n 0) (mu 0) (m2 0) (heaven 1))
```

### make-num

Constructors come in two layers. `%make-num` is the raw
allocator generated by `defstruct`. `make-num` wraps it:
inspect the last character of `txt` and set `heaven` to 0
when the column name ends in `-`. Same wrapping pattern
appears later in `make-cols`.

```lisp
(defun make-num (&rest args
                 &aux (i (apply #'%make-num args)))
  (setf $heaven (if (eq #\- (ch $txt -1)) 0 1))
  i)
```

## Operations on atoms

Every column supports four verbs: `add`, `mid`, `spread`,
`norm`. Each verb has two methods, one per kind, dispatched
by CLOS. Pairs are shown adjacently so the symmetry shows.

### add-sym

`add` updates a summary by one observation. For `sym` it
either bumps an existing cell or pushes a fresh one. The
weight `w` defaults to 1; passing `-1` reverses an earlier
add. Missing values (the symbol `?`) are skipped: every
later step assumes a column's `n` reflects only seen
values.

```lisp
(defmethod add ((i sym) v &optional (w 1))
  (unless (eql v '?)
    (incf $n w)
    (let ((cell (assoc v $has :test #'equal)))
      (if cell
          (incf (cdr cell) w)
          (push (cons v w) $has))))
  v)
```

### add-num

The `num` add applies Welford's two-line update. `mu`
moves toward the new value at rate `1/n`. `m2` then uses
the *new* `mu` for its own increment; that is the identity
that lets `sd` be recovered later. The `(minusp w) (<= $n
2)` clause resets to zero when subtracting from a
near-empty summary; the on-line variance update is
unstable below `n=2`.

```lisp
(defmethod add ((i num) v &optional (w 1))
  (unless (eq v '?)
    (cond ((and (minusp w) (<= $n 2))
           (setf $n 0 $mu 0 $m2 0))
          (t (incf $n w)
             (let ((delta (- v $mu)))
               (incf $mu (float (/ (* w delta) $n)))
               (incf $m2 (float
                          (* w delta (- v $mu))))))))
  v)
```

### mid-sym

`mid` returns a central tendency. For symbols this is the
mode: the value with the largest count.

```lisp
(defmethod mid ((i sym))
  (labels ((most (a b)
             (if (> (cdr a) (cdr b)) a b)))
    (car (reduce #'most $has))))
```

### mid-num

For numbers `mid` is just the running mean. Same name,
same role; only the body changes.

```lisp
(defmethod mid ((i num)) $mu)
```

### spread-sym

`spread` returns a width. For symbols this is the Shannon
entropy of `has`, in bits.

```lisp
(defmethod spread ((i sym))
  (loop for (_ . v) in $has
        for p = (/ v $n)
        sum (- p (log p 2))))
```

### spread-num

For numbers `spread` is the standard deviation, recovered
from `m2` and `n`. Both methods report zero on
under-populated summaries. Because `mid` and `spread`
take the same argument shape regardless of column type,
`dist` (below) can loop over a heterogeneous list of
columns without checking which is which.

```lisp
(defmethod spread ((i num))
  (if (< $n 2) 0
      (sqrt (/ (max 0 $m2) (1- $n)))))
```

### norm-sym

`norm` maps a value to `[0,1]`. The symbol method is the
identity; categorical values have no natural ordering to
normalise against.

```lisp
(defmethod norm ((i sym) v) v)
```

### norm-num

The number method computes a clamped z-score, then squashes
it through a logistic with slope 1.7 (the standard
approximation to the gaussian CDF). Output reads as
"fraction of column at or below `v`".

```lisp
(defmethod norm ((i num) v)
  (if (eq v '?) v
      (let ((z (/ (- v $mu) (+ (spread i) 1e-32))))
        (/ 1 (+ 1 (exp (* -1.7
                          (max -3 (min 3 z)))))))))
```

```
0.005

?
```

## Folds

### adds and sub

Two folds round out the atom layer. `adds` walks a list
and folds each value into a summary via `add`; the default
summary is a fresh `num`. `sub` is the symmetric remove
(weight `-1`). Together they make summaries reversible,
which the active learner uses to shuffle rows between
piles without recomputing means from scratch.

```lisp
(defun adds (lst &optional (summary (make-num)))
  "Fold LST into SUMMARY via `add`. Returns SUMMARY."
  (dolist (v lst summary)
    (add summary v)))

(defun sub (it v)
  "Remove V from IT (symmetric with `add`)."
  (add it v -1))
```

```

#S(NUM :N 5 :MU 3.0 :M2 10.0 :HEAVEN 1 ...)
[26]> (mid (adds '(1 2 3 4 5)))
3.0
[27]> (sub n 50) (sub n 40)

(3 20.0)
```

## Containers

### cols

A `cols` object is a triple of column lists carved from
one header row. `all` is every column; `x` is input
features; `y` is goals; `names` keeps the original header
strings for cloning.

```lisp
(defstruct (cols (:constructor %make-cols))
  names x y all)
```

### data

A `data` object is a list of `rows` plus the `cols` that
summarise them. The `mid` slot caches the centroid (one
`mid` per column); mutation via `add` invalidates it by
setting it to nil.

```lisp
(defstruct (data (:constructor %make-data))
  rows cols mid)
```

### make-cols

`make-cols` walks the header strings and instantiates the
right kind of column for each. The local `ako` ("a-kind-of")
chooses `make-num` when the name starts uppercase, else
`make-sym`. The suffix character routes the column into
`y` (when it ends in `!`, `-`, `+`), into `x` (otherwise),
or drops it entirely (when it ends in `X`).

```lisp
(defun make-cols (txts
                  &aux (i (%make-cols :names txts))
                       (n -1))
  (labels ((ako (txt)
             (if (upper-case-p (ch txt 0))
                 #'make-num #'make-sym))
           (one (txt)
             (funcall (ako txt) :txt txt :at (incf n)))
           (end (col) (ch (? col txt) -1)))
    (setf $all (mapcar #'one txts))
    (dolist (col $all i)
      (unless (eql (end col) #\X)
        (if (find (end col) "!-+")
            (push col $y)
            (push col $x))))))
```

### make-data

`make-data` assumes the first row of input is the header.
Builds the `cols`, then folds the remaining rows via
`adds`. Result: a fully populated `data` object.

```lisp
(defun make-data (&optional rows)
  (adds (cdr rows)
        (%make-data :cols (make-cols (car rows)))))
```

### add-cols

`add` extends to containers by delegation. The `cols`
method maps `add` across each column with the matching
cell of `row`.

```lisp
(defmethod add ((i cols) row &optional (w 1))
  (mapcar (lambda (col v) (add col v w)) $all row)
  row)
```

### add-data

The `data` method threads through `cols`, then either
pushes the row (positive weight) or removes it (negative
weight), invalidating the centroid cache. The shared verb
`add` now works uniformly on all four kinds: `sym`, `num`,
`cols`, `data`.

```lisp
(defmethod add ((i data) row &optional (w 1))
  (setf $mid nil)
  (add $cols row w)
  (if (plusp w)
      (push row $rows)
      (setf $rows (remove row $rows
                          :test #'equal :count 1)))
  row)
```

### mid-data

`mid` for `data` returns the centroid: a list with one
`mid` per column, memoised in `$mid`. Many distance calls
compare a row against the centroid, so caching pays off
quickly.

```lisp
(defmethod mid ((i data))
  (or $mid
      (setf $mid
            (mapcar #'mid (? i cols all)))))
```

## Distance

### dist

`dist` rolls up a list of per-cell costs into one scalar
via a Minkowski norm. The exponent `p` is read from the
config. `f` is the per-element cost function supplied by
the caller.

```lisp
(defun dist (lst f &aux (d 0))
  (dolist (v lst
           (expt (/ d (length lst)) (/ 1 !p)))
    (incf d (expt (funcall f v) !p))))
```

### disty

`disty` is the goal distance: how far a row sits from the
ideal goal vector. For each y-column, take the absolute
gap between its normalised value and `heaven`. Minkowski
collapses those gaps into one number used everywhere as
the row's "y-score". Lower is better.

```lisp
(defun disty (data row
              &optional (cols (? data cols y)))
  (dist cols
        (lambda (i)
          (abs (- (norm i (elt row $at)) $heaven)))))
```

### distx

`distx` is the feature-space distance between two rows.
The `data` dispatcher loops over x-columns and asks each
cell pair for its cost; missing-on-both costs 1 (worst
case, so unknowns do not lie). Symbols compare by
equality. Numbers compare by absolute difference of
normalised values. When only one side is missing, the
unknown is pushed to the extreme furthest from the known:
pessimistic imputation that keeps the dimension paying
its weight.

```lisp
(defmethod distx ((i data) r1 r2)
  (dist (? i cols x)
        (lambda (i)
          (let ((a (elt r1 $at))
                (b (elt r2 $at)))
            (if (and (eq a '?) (eq b '?))
                1 (distx i a b))))))

(defmethod distx ((i sym) a b)
  (if (equal a b) 0 1))

(defmethod distx ((i num) a b
                  &aux (a (norm i a)) (b (norm i b)))
  (cond ((eq a '?)
         (abs (- b (if (> b 0.5) 0 1))))
        ((eq b '?)
         (abs (- a (if (> a 0.5) 0 1))))
        (t (abs (- a b)))))
```

## Trees

### tree-struct

A `tree` node carries a cloned `data` (so summaries are
local to the node), a `ynum` summarising the disty of its
rows, the chosen split column `col` and `cut`, the two
children, and a display string `hdr`. Leaves have null
`left` and `right`.

```lisp
(defstruct (tree (:copier nil))
  d ynum col cut left right hdr)
```

### branch-and-cuts

Splits dispatch on column kind through two tiny generic
functions. `branch?` says whether a value goes left under
a cut: equality for symbols, `<=` for numbers. `%cuts`
returns candidate cut values: every distinct symbol, or
just the median for numbers. The median-only choice for
numbers is regularisation; with ~50 labels, trying every
threshold would overfit.

```lisp
(defmethod branch? ((c sym) v cut) (equal v cut))
(defmethod branch? ((c num) v cut) (<= v cut))

(defmethod %cuts ((c sym) vs)
  (remove-duplicates vs :test #'equal))

(defmethod %cuts ((c num) vs)
  (list (elt (sort vs #'<) (floor (length vs) 2))))
```

### new-tree

`new-tree` builds a node before any splitting decision.
Clone the data, summarise the disty of its rows into
`ynum`, attach a header string. If `tree-grow` later
finds a useful split, it fills in `col`, `cut`, and
children; otherwise the node stays a leaf.

```lisp
(defun new-tree (d rs &optional (hdr ""))
  "Tree node: cloned data, a num of distys, header."
  (make-tree
    :d (make-data (cons (? d cols names) rs))
    :ynum (adds (mapcar (lambda (r) (disty d r)) rs)
                (make-num))
    :hdr hdr))
```

### tree-grow

The recursion. Build a node. If too few rows remain, stop.
Otherwise score every legal split, take the lowest, and
recurse on each side. The base case keeps recursion finite
without an explicit depth bound; the data shrinks every
level. Helpers `go-left?`, `tree-cuts`, `tree-split`,
`tree-splits`, `kid-hdr` factor the work.

```lisp
(defun go-left? (c v cut)
  (or (eq v '?) (branch? c v cut)))

(defun tree-cuts (c rs)
  (let ((vs (loop for r in rs
                  for v = (elt r (? c at))
                  unless (eq v '?) collect v)))
    (and vs (%cuts c vs))))

(defun tree-split (d c cut rs)
  (let (l r (ln (make-num)) (rn (make-num)))
    (dolist (row rs)
      (if (go-left? c (elt row (? c at)) cut)
          (progn (push row l) (add ln (disty d row)))
          (progn (push row r) (add rn (disty d row)))))
    (list (+ (* (? ln n) (spread ln))
             (* (? rn n) (spread rn)))
          c cut l r)))

(defun tree-splits (d rs leaf)
  (loop for c in (? d cols x)
        append (loop for cut in (tree-cuts c rs)
                     for s = (tree-split d c cut rs)
                     when (>= (min (length (fourth s))
                                   (length (fifth s)))
                              leaf)
                     collect s)))

(defun kid-hdr (c cut left?)
  (format nil "~a ~a ~a" (? c txt)
          (if left?
              (if (typep c 'sym) "==" "<=")
              (if (typep c 'sym) "!=" ">"))
          cut))

(defun tree-grow (d rs &optional (leaf 3) (hdr ""))
  (let ((t0 (new-tree d rs hdr)))
    (when (>= (length rs) (* 2 leaf))
      (let ((splits (tree-splits d rs leaf)))
        (when splits
          (let* ((best (extremum-by splits #'first #'<))
                 (c (second best)) (cut (third best)))
            (setf (? t0 col) c (? t0 cut) cut
                  (? t0 left)
                   (tree-grow d (fourth best) leaf
                              (kid-hdr c cut t))
                  (? t0 right)
                   (tree-grow d (fifth best) leaf
                              (kid-hdr c cut nil)))))))
    t0))
```

### tree-leaf

Routes a single row through a built tree to its terminal
node.

```lisp
(defun tree-leaf (t0 r)
  (if (null (? t0 left)) t0
      (tree-leaf
        (if (go-left?
              (? t0 col)
              (elt r (? (? t0 col) at))
              (? t0 cut))
            (? t0 left) (? t0 right))
        r)))
```

### tree-show

Walks the tree depth-first, one node per line: `mu` and
`n` on the left, indent and condition on the right.

```lisp
(defun tree-show (t0 &optional (lvl 0))
  (format t "~&~4,2f (~3d)   ~{~A~}~a~%"
          (mid (? t0 ynum)) (? (? t0 ynum) n)
          (loop repeat (max 0 (1- lvl))
                collect "|.. ")
          (? t0 hdr))
  (when (? t0 left)
    (tree-show (? t0 left)  (1+ lvl))
    (tree-show (? t0 right) (1+ lvl))))
```

## Active learning

### wins

`wins` returns a closure that scores a row 0..100 against
a "god" data set: 100 means as close to the best known
disty as possible, 50 means around the median. Used only
for evaluation; the learner never sees these numbers
during training.

```lisp
(defun wins (god)
  (let* ((ys (mapcar (lambda (r) (disty god r))
                     (? god rows)))
         (ds (sort ys #'<))
         (lo (first ds))
         (md (elt ds (floor (length ds) 2))))
    (lambda (r)
      (floor (* 100
                (- 1 (/ (- (disty god r) lo)
                        (+ (- md lo) 1e-32))))))))
```

### extremum-by

`extremum-by` returns argmin or argmax in one O(n) pass.
The comparator decides direction: `#'<` gives argmin under
the key. Pulling this out lets later code stay readable
without sorting the whole list when it only needs the
head.

```lisp
(defun extremum-by (lst key cmp)
  "Argmin or argmax of LST under KEY, ordered by CMP."
  (let* ((best (car lst)) (m (funcall key best)))
    (dolist (x (cdr lst) best)
      (let ((k (funcall key x)))
        (when (funcall cmp k m)
          (setf best x m k))))))
```

### rebalance

`rebalance` keeps the `best` pile small. Whenever it
exceeds `sqrt(|labels|)`, the worst-disty row in `best`
is moved to `rest`. The sqrt cap is empirically the sweet
spot: small enough that the centroid stays
discriminating, big enough that one noisy row cannot
poison it.

```lisp
(defun rebalance (best rest lab)
  (when (> (length (? best rows))
           (sqrt (length (? lab rows))))
    (let ((bad (extremum-by (? best rows)
                            (lambda (r) (disty lab r))
                            #'>)))
      (sub best bad) (add rest bad))))
```

### closer

The acquisition test: would labelling row `r` push it
nearer the centroid of `best` than that of `rest`?
Centroid distance is the cheapest signal that still uses
everything we know about both piles.

```lisp
(defun closer? (lab best rest r)
  (< (distx lab r (mid best))
     (distx lab r (mid rest))))
```

### warm-start

Pays for the first four labels, then splits them into
`best` (top sqrt by disty) and `rest` (the others). This
bootstraps `best` and `rest` with enough mass for
`closer?` to mean something on the next row.

```lisp
(defun warm-start (rs lab best rest ys
                   &aux (start 4))
  (dotimes (_ (min start (length rs)))
    (let ((r (pop rs)))
      (add lab r) (add ys (disty lab r))))
  (let* ((sorted (sort (copy-list (? lab rows)) #'<
                       :key (lambda (r) (disty lab r))))
         (n (max 1 (floor (sqrt (length sorted))))))
    (dolist (r (subseq sorted 0 n)) (add best r))
    (dolist (r (subseq sorted n))   (add rest r)))
  rs)
```

### active-loop

The main loop. Shuffle the pool; warm-start; walk the
capped pool. For each row, if the acquisition test fires,
pay for the label, fold it into `best`, and rebalance.
Stop when `ys` (the num counting label spend) hits the
budget. Returns three values: `best`, `lab`, label count.

```lisp
(defun active (rs &aux (hd (list (car rs))))
  (let* ((lab  (make-data hd))
         (best (make-data hd))
         (rest (make-data hd))
         (ys   (make-num))
         (pool (warm-start (shuffle (cdr rs))
                           lab best rest ys)))
    (loop for r in (subseq pool 0
                           (min !few (length pool)))
          while (< (? ys n) !budget) do
          (when (closer? lab best rest r)
            (add ys (disty lab r))
            (add lab r) (add best r)
            (rebalance best rest lab)))
    (values best lab (? ys n))))
```

### validate

The holdout evaluator. Half the data trains via `active`.
A tree is grown on the labelled rows. Test rows are ranked
by their leaf's mean disty; the top `check` are inspected
against god truth. Returns three numbers: score of best
train row, score of best test pick, total label spend.

```lisp
(defun validate (rs god w &optional (check 5))
  (let* ((body (shuffle (cdr rs)))
         (n (floor (length body) 2))
         (train (cons (car rs) (subseq body 0 n)))
         (test  (subseq body n)))
    (multiple-value-bind (best lab labels)
        (active train)
      (declare (ignore best))
      (let* ((y-god (lambda (r) (disty god r)))
             (train-best (extremum-by (? lab rows)
                                      y-god #'<))
             (tr (tree-grow lab (? lab rows)))
             (ranked
              (sort (copy-list test) #'<
                    :key (lambda (r)
                           (mid (? (tree-leaf tr r)
                                   ynum)))))
             (top (subseq ranked 0
                          (min check (length ranked))))
             (pick (extremum-by top y-god #'<)))
        (values (funcall w train-best)
                (funcall w pick)
                (+ labels check))))))
```

```
[43]> (defparameter god (make-data rs))
GOD
[44]> (defparameter w (wins god))
W
[45]> (multiple-value-bind (tr ho lab) (validate rs god w)
        (format t "~&train=~a hold=~a labels=~a" tr ho lab))
train=92 hold=88 labels=55
NIL
```

## Examples

Each `eg--` is a runnable demonstration of one concept
introduced above. Functions print to stdout and own their
own state. Names map to CLI flags through the `lib`
dispatcher: `--rows` runs `eg--rows`, and so on.

```lisp
(defun %stride (rows key &optional (n 30))
  (loop for x in (sort rows #'< :key key)
        by (lambda (l) (nthcdr n l))
        do (print x)))

(defun eg--the (_)         (print *the*))

(defun eg--rows (&optional (file !file))
  (print (subseq (read-csv file) 380)))

(defun eg--data (&optional (file !file))
  (print (? (make-data (read-csv file)) cols y)))

(defun eg--stats (&optional (file !file))
  (let ((i (make-data (read-csv file))))
    (dolist (c (? i cols all))
      (format t "~&~12a  mid=~a  spread=~a~%"
              (? c txt) (mid c) (spread c)))))

(defun eg--norm (&optional (file !file))
  (let* ((i (make-data (read-csv file)))
         (r (car $rows)))
    (dolist (c (? i cols y))
      (let ((v (elt r (? c at))))
        (format t "~&~8a  raw=~a  norm=~a~%"
                (? c txt) v (norm c v))))))

(defun eg--ydata (&optional (file !file))
  (let ((i (make-data (read-csv file))))
    (%stride $rows (lambda (r) (disty i r)))))

(defun eg--xdata (&optional (file !file))
  (let* ((i (make-data (read-csv file)))
         (r0 (car $rows)))
    (%stride $rows (lambda (r) (distx i r0 r)))))

(defun eg--tree (&optional (file !file))
  (let ((rs (read-csv file)))
    (multiple-value-bind (best lab labels) (active rs)
      (declare (ignore best))
      (format t "~&;; labels used: ~d~%" labels)
      (tree-show (tree-grow lab (? lab rows))))))

(defun eg--active (&optional (file !file))
  (let* ((rs (read-csv file))
         (god (make-data rs))
         (w (wins god)))
    (loop repeat 20 do
          (multiple-value-bind (train hold labels)
              (validate rs god w)
            (format t "~&~3d ~3d ~3d~%"
                    train hold labels)))))

(defun eg--all (&optional (arg !file))
  (do-symbols (s *package*)
    (unless (eq s 'eg--all) (run s arg))))
```

## Tutorial

A short tour of the Common Lisp constructs the body uses.
Readers fluent in C, Python, or Java recognise most of the
structure; the items below are spots where Lisp diverges
and a fresh reader is most likely to stumble.

### Quote vs sharp-quote: 'x vs sharp-x

`'x` is data: it stops evaluation and returns the symbol
or list literally. `#'x` is the function value of `x`,
used wherever a function is passed as an argument. So
`'foo` and `#'foo` are different things. First seen in
[extremum-by](#extremum-by) where comparators arrive as
`#'<`, and in the `*the*` quoted a-list at [the](#the).

```
[T1]> 'foo
FOO
[T2]> #'<
#<FUNCTION <>
```

### nil is empty list and false

No separate boolean type. The empty list `'()`, the
symbol `nil`, and false are the same value. Anything else
is true. So `(if (null xs) ...)` and `(if xs ...)` read
naturally on lists. Used throughout.

### let, let-star, double-paren bindings

`let` binds variables in parallel; `let*` binds
sequentially. Bindings live inside an extra pair of
parens: `(let ((x 1) (y 2)) ...)`. The inner parens
surround each `(name value)`. `let*` shows up wherever a
later binding refers to an earlier one (e.g.
[validate](#validate)).

```
[T3]> (let  ((a 1) (b 2)) (+ a b))
3
[T4]> (let* ((a 1) (b (* a 2))) b)
2
```

### setf as universal place setter

`(setf place value)` writes `value` into `place`. The
"place" can be a variable, a slot, an array cell, an
a-list entry. `(setf $mu 0)` is the slot write inside
[add-num](#add-num); `(setf (cdr cell) v)` is the a-list
write inside [add-sym](#add-sym).

### Lambda lists: optional, key, rest, aux

`&optional` introduces defaults (`(w 1)` in
[add-num](#add-num)). `&key` introduces named arguments
(`make-num :txt "x"`). `&rest` collects extras into a
list ([make-num](#make-num)). `&aux` declares locals
initialised after parameters but before the body, used
in [make-num](#make-num) to inspect `txt`.

```
[T5]> (defun f (x &optional (y 0) &aux (z (* x 2)))
        (+ x y z))
F
[T6]> (f 3)
9
[T7]> (f 3 1)
10
```

### Multiple values: values and multiple-value-bind

A function returns more than one value with `values`; the
caller destructures with `multiple-value-bind`. Used by
[active-loop](#active-loop) to return three things at
once and by [validate](#validate) to read them.

```
[T8]> (defun two () (values 1 2))
TWO
[T9]> (multiple-value-bind (a b) (two) (list a b))
(1 2)
```

### defstruct and slot access

`defstruct` declares a record type and generates a
constructor, accessors, and a type predicate. The
`(:constructor %make-num)` clause renames the auto
constructor so we can wrap it in [make-num](#make-num).
`(slot-value i 'mu)` reads the `mu` slot of `i`; the
reader macro `$mu` from `lib.lisp` expands to exactly
that when `i` is in scope.

### defmethod and CLOS dispatch

`defmethod name ((arg type) ...)` defines one method of
a generic function. The runtime picks the method whose
argument types match. The body has parallel methods
(`add`, `mid`, `spread`, `norm`) for `sym` and `num`;
the dispatcher chooses without an explicit type test in
the caller.

### loop: a mini-language inside Lisp

`loop` accepts a small DSL of clauses: `for`, `in`,
`collect`, `sum`, `when`, `do`. Every loop in the body
uses some subset; the most exotic is in
[tree-show](#tree-show)'s `loop repeat n collect "|.. "`
for indenting.

```
[T10]> (loop for x in '(1 2 3) collect (* x x))
(1 4 9)
```

### labels for local recursion

`labels` is `let` for functions: it introduces local
defuns, possibly mutually recursive. [make-cols](#make-cols)
uses it to define `ako`, `one`, and `end` without
polluting the top level.

```
[T11]> (labels ((sq (x) (* x x))) (sq 5))
25
```

### dolist and the implicit return value

`(dolist (var list result) body)` walks `list` binding
`var`. The optional `result` form is evaluated and
returned at the end. [make-cols](#make-cols) uses this to
return `i` after pushing each column into its bucket.

```
[T12]> (dolist (x '(1 2 3) 'done) (print x))
1
2
3
DONE
```

### cons, list, append, push, pop

`cons` adds one head to a list; `list` builds a fresh
list from arguments; `append` glues lists end to end.
`push` and `pop` are macros that mutate the variable
holding the list. `(push x xs)` is equivalent to
`(setf xs (cons x xs))`. Used everywhere; especially in
[add-sym](#add-sym).

### Macro vs function

A macro receives unevaluated source forms and returns
new source forms; the compiler then evaluates the result.
The reader macros `!`, `$`, and `?` from `lib.lisp` are
macros expanding to `(second (assoc ...))` and
`(slot-value ...)` calls. They look like variables in
the source but compile to ordinary code.

### Reader macros: hash-pipe, hash-backslash, sharp-quote

`#|...|#` opens a block comment. `#\X` is a character
literal (e.g. `#\-` in [make-num](#make-num)). `#'name`
is sharp-quote (function value).

### declare ignore

When a `multiple-value-bind` binds a value the body does
not use, declare it ignored to silence the compiler:
`(declare (ignore best))` in [validate](#validate).
