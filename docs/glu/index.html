A Common Lisp dialect
that fits in your head.

glu is six macros and one rule: the subject is implicit. fri2.lisp — a complete active-learning system — is written in it. 

;;; squared odds, fed into a Welford summary

(glu stat "Online mean+m2 (Welford)."
    ((n 0) (mu 0) (m2 0))

  (add (v)
    "Fold V into running stats."
    (incf $n)
    (let ((d (- v $mu)))
      (incf $mu (/ d $n))
      (incf $m2 (* d (- v $mu))))))

(let ((i (make-stat)))
  (for/ (add i (* x x)) for x in '(1 2 3 4 5)
                            if (oddp x))
  (format t "n=~a mean=~,2f~%" $n $mu))
;; => n=3 mean=11.67  (squares of 1, 3, 5: 1+9+25 = 35; 35/3)

One glu form — struct, docstring, method. One for/ — filter, transform, fold into the stats in a single pass. $n reads as (slot-value i 'n) and works under incf. The full sym+num pair with shared add and mid is fifteen lines in glu versus forty in raw CL.

Why glu?

Tiny

Six macros, ~50 lines of definitions. Drop into any CL file: (load "glu.lisp"). No package manager, no asdf, no build system.

Anaphoric

In a method, i is always self. In an aif, it is always the test. No boilerplate let-bindings.

Composable

$ and ? read as Lisp forms. They setf, incf, nest — invisible until you look.

What it replaces

Three short comparisons against plain Common Lisp and Python. Every Lisp snippet on this page has been load-tested in SBCL; outputs are real.

List comprehension

Squares of odd numbers in '(1 2 3 4 5):

Plain CL loop

(loop for x in lst
      when (oddp x)
      collect (* x x))
;; => (1 9 25)

glu for/

(for/ (* x x)
       for x in lst
       if (oddp x))
;; => (1 9 25)

Python

[x*x
 for x in lst
 if x % 2]
# => [1, 9, 25]

Class with three methods

A streaming numeric summary — count, mean, sd:

Plain CL

(defstruct num
  "Welford summary."
  (n 0) (mu 0) (m2 0))

(defmethod add ((s num) v)
  "Fold V."
  (incf (slot-value s 'n))
  (let ((d (- v (slot-value s 'mu))))
    (incf (slot-value s 'mu)
          (/ d (slot-value s 'n)))
    (incf (slot-value s 'm2)
          (* d (- v (slot-value s 'mu))))))

(defmethod mid ((s num))
  "Mean."
  (slot-value s 'mu))

(defmethod spread ((s num))
  "SD."
  (if (< (slot-value s 'n) 2) 0
      (sqrt (/ (slot-value s 'm2)
               (1- (slot-value s 'n))))))

glu

(glu num "Welford summary."
    ((n 0) (mu 0) (m2 0))

  (add (v)
    "Fold V."
    (incf $n)
    (let ((d (- v $mu)))
      (incf $mu (/ d $n))
      (incf $m2 (* d (- v $mu)))))

  (mid ()
    "Mean."
    $mu)

  (spread ()
    "SD."
    (if (< $n 2) 0
        (sqrt (/ $m2 (1- $n))))))

Python

class Num:
    """Welford summary."""

    def __init__(self):
        self.n  = 0
        self.mu = 0
        self.m2 = 0

    def add(self, v):
        """Fold V."""
        self.n += 1
        d = v - self.mu
        self.mu += d / self.n
        self.m2 += d * (v - self.mu)

    def mid(self):
        """Mean."""
        return self.mu

    def spread(self):
        """SD."""
        if self.n < 2:
            return 0
        return (self.m2 / (self.n - 1)) ** 0.5

Twenty-two lines of plain CL, twelve of glu, twenty of Python. The cost of CL was the noise around slot-value and the split between struct and methods — not the language itself. glu removes both.

The macros

Ordered by impact, top to bottom: bundle, comprehend, slot, chain, branch.

(glu name [doc] slots methods...)

Struct + methods, in one form.

Bundles a defstruct with its defmethods. The first parameter is always i, so $slot just works in every method body.

Optional second arg is the struct docstring. The macro always emits (:constructor %make-NAME) (private) plus make-NAME (public). If the pseudo-method (:make args body) is provided, its body becomes the public factory; otherwise make-NAME is a thin (apply #'%make-NAME args) wrapper. (:opts ((opt ...) ...)) adds extra defstruct options when needed.

(glu num "Numeric column with Welford stats."
    ((n 0) (mu 0) (m2 0))

  (add (v)
    "Fold V into running stats."
    (incf $n)
    (let ((d (- v $mu)))
      (incf $mu (/ d $n))
      (incf $m2 (* d (- v $mu)))))

  (mid ()
    "Return the running mean."
    $mu)

  (spread ()
    "Return the running sd."
    (if (< $n 2) 0
        (sqrt (/ $m2 (1- $n))))))

Trick. Same method name across classes? Use the same name — CL dispatches on the first arg's type. (add s v) hits sym's body or num's depending on s.

Reopen. Add methods to an existing class with (glu+ NAME METHODS...). glu forwards its method list to glu+. glu+ rejects :make — the constructor is a struct-time concern, defined once by glu.

(glu+ num
  (twice ()
    "Return 2*mu."
    (* 2 $mu)))

(for/ expr for x in lst if test ...)

List comprehension.

Reads like Python. Walks a sequence of for and if clauses, collecting expr for each iteration that survives the filters. Nil results are skipped — a guard expression doubles as a filter.

(for/ (* x x) for x in '(1 2 3 4 5) if (oddp x))
;; => (1 9 25)

;; nested loops, multiple filters
(for/ (list x y)
       for x in '(1 2 3)
       for y in '(a b)
       if (oddp x))
;; => ((1 A) (1 B) (3 A) (3 B))

;; nil-skipping is a free filter
(for/ (and (oddp x) x) for x in '(1 2 3 4 5))
;; => (1 3 5)   ; even values produce nil, dropped

$field

Self slot access.

Inside a method whose first parameter is named i — glu methods always do — $field expands to (slot-value i 'field). setf-able, incf-able, push-able.

(defmethod mid ((i num))
  "Return the running mean."
  $mu)

(defmethod add ((i sym) v &optional (w 1))
  "Fold V into symbol counts."
  (unless (eql v '?)
    (incf $n w)))

Trick. Outside a method, just bind i in a let and the shortcut still works:

(let ((i (make-stat)))
  (for/ (add i v) for v in lst)
  (format t "mean=~a~%" $mu))

(? x a b c)

Nested slot access.

Chains slot-value calls. setf-able. Replaces the noisy (slot-value (slot-value ...)) tower.

(? data cols y)
;; ≡  (slot-value (slot-value data 'cols) 'y)

;; setf works through the chain
(setf (? tree left) new-left)

(aif test then else)

Anaphoric if.

Binds the test result to it in both branches. From Paul Graham's On Lisp.

(aif (find flag opts :key #'third :test #'equalp)
     (setf (second it) (thing arg)))

;; common idiom: search-and-act
(aif (gethash key cache)
     it
     (setf (gethash key cache) (compute key)))

(fn ...) (fnn ...) (fnnn ...)

Lambda shortcuts.

One-, two-, three-arg lambdas with implicit parameters _, __, ___. Kills the (lambda (x) ...) overhead in mapcar, sort :key, remove-if, etc.

(mapcar (fn  (* _ _))      '(1 2 3))
;; => (1 4 9)

(mapcar (fnn (+ _ __))     '(1 2 3) '(10 20 30))
;; => (11 22 33)

(mapcar (fnnn (list _ __ ___))
                            '(a b) '(1 2) '(x y))
;; => ((A 1 X) (B 2 Y))

Trick. Pair with $slot when the inner lambda's first arg is a struct named i — but inside fn the arg is _, so use the explicit form for that case.

The everything example

Two columns — symbolic and numeric — with shared method names add and mid (CL dispatches by type), plus spread for num. Both versions load, both print sym mid=RED num mid=3 sd=1.58.

Plain Common Lisp

(defstruct sym
  "Symbol column: count, mode."
  (n 0) (has nil))

(defstruct num
  "Numeric column: count, mean, sd."
  (n 0) (mu 0) (m2 0))

(defmethod add ((s sym) v)
  "Fold V; track frequency."
  (incf (slot-value s 'n))
  (let ((cell (assoc v (slot-value s 'has))))
    (if cell
        (incf (cdr cell))
        (push (cons v 1) (slot-value s 'has)))))

(defmethod add ((s num) v)
  "Fold V into running mean+m2."
  (incf (slot-value s 'n))
  (let ((d (- v (slot-value s 'mu))))
    (incf (slot-value s 'mu)
          (/ d (slot-value s 'n)))
    (incf (slot-value s 'm2)
          (* d (- v (slot-value s 'mu))))))

(defmethod mid ((s sym))
  "Mode."
  (car (reduce (lambda (a b)
                 (if (> (cdr a) (cdr b)) a b))
               (slot-value s 'has))))

(defmethod mid ((s num))
  "Mean."
  (slot-value s 'mu))

(defmethod spread ((s num))
  "SD."
  (if (< (slot-value s 'n) 2) 0
      (sqrt (/ (slot-value s 'm2)
               (1- (slot-value s 'n))))))

glu

(glu sym "Symbol column: count, mode."
    ((n 0) (has nil))

  (add (v)
    "Fold V; track frequency."
    (incf $n)
    (let ((cell (assoc v $has)))
      (if cell
          (incf (cdr cell))
          (push (cons v 1) $has))))

  (mid ()
    "Mode."
    (car (reduce (lambda (a b)
                   (if (> (cdr a) (cdr b)) a b))
                 $has))))

(glu num "Numeric column: count, mean, sd."
    ((n 0) (mu 0) (m2 0))

  (add (v)
    "Fold V into running mean+m2."
    (incf $n)
    (let ((d (- v $mu)))
      (incf $mu (/ d $n))
      (incf $m2 (* d (- v $mu)))))

  (mid ()
    "Mean."
    $mu)

  (spread ()
    "SD."
    (if (< $n 2) 0
        (sqrt (/ $m2 (1- $n))))))

Run them both:

(let ((s (make-sym)) (n (make-num)))
  (for/ (add s v) for v in '(red blue red green red blue))
  (for/ (add n v) for v in '(1 2 3 4 5))
  (format t "sym mid=~a   num mid=~a sd=~,2f~%"
          (mid s) (mid n) (spread n)))
;; => sym mid=RED   num mid=3 sd=1.58

Two structs become two glu blocks. Six defmethod heads — each repeating the type tag and the slot-value machinery — vanish into the bundling. Every (slot-value s 'X) shrinks to $X. Same Lisp. Less of it. Same generic dispatch — add on a sym and add on a num stay distinct methods on the same generic.

Try it

Two files. No build system, no package manager. If you have CLISP or SBCL, you have glu.

$ git clone https://github.com/timm/lisp
$ cd lisp
$ sbcl --script fri2.lisp --stats
$ clisp fri2.lisp --tree

glu.lisp is the dialect. fri2.lisp loads it and adds a complete active-learning system — CSV loader, online column stats, decision-tree builder, train/test loop — and serves as the running example for everything on this page.
